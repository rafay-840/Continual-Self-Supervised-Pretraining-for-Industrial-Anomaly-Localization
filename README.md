## Overview

Industrial anomaly detection must identify defects using **only defect-free training images**, since real manufacturing defects are rare, heterogeneous, and expensive to annotate. This project investigates two open questions at the intersection of self-supervised learning and continual learning:

> **RQ1:** Can SimCLR pretraining on small (~200 images), unlabeled, in-domain datasets produce better anomaly detection features than ImageNet pretraining or random initialization?

> **RQ2:** When new product categories arrive sequentially, which continual learning strategy best preserves anomaly detection performance on previously-learned categories?

We combine **SimCLR** contrastive pretraining with **PatchCore** nearest-neighbor anomaly scoring on three MVTec AD categories, then benchmark four continual learning strategies using **Backward Transfer (BWT)** as the primary forgetting metric.

---

## Key Findings

| Finding | Result |
|---|---|
| SimCLR-300ep vs. Scratch | ✅ Outperforms on **all 3 categories** |
| SimCLR-100ep vs. Scratch | ❌ Insufficient — training budget is the binding constraint |
| Best CL strategy | **Coreset Replay** — reduces forgetting by **83%** (BWT: −0.011 vs −0.063) |
| Worst CL strategy | **PackNet** — *underperforms* the naive baseline due to capacity starvation |
| Structural limitation found | Absence-type defects (missing wires/cables) are undetectable by local patch methods |

## Running Experiments

Run notebooks **in order**. Each is fully self-contained and safe to re-run after Colab disconnects.

### Phase 1 — Pretraining + Evaluation

```
Notebook 1  →  Notebook 2 (run for each category)  →  Notebook 3
```

**Notebook 1 — Data Pipeline**
- Downloads and extracts MVTec AD from Drive
- Implements `MVTecDataset` class with train/test splits and defect mask loading
- Verifies category counts and pixel-level annotation overlays

**Notebook 2 — SimCLR Pretraining**

Key parameters to set per run:
```python
CATEGORY    = "bottle"    # also: "cable", "leather"
NUM_EPOCHS  = 300         # use 100 for ablation study
BATCH_SIZE  = 64
LR          = 3e-4
TEMPERATURE = 0.5         # τ in NT-Xent loss
```

> ⚠️ **Leather uses special augmentation settings** (saved as tag `latest_v3_300ep`):
> - `HUE_JITTER = 0` — hue shift is itself a valid defect signal in texture categories
> - `CROP_SCALE_MIN = 0.2` — forces genuinely local grain crops to create harder negatives

Checkpoints save to Drive after each epoch. Training is fully resumable.

**Notebook 3 — PatchCore Evaluation**

Evaluates all 4 backbone strategies (Scratch, ImageNet, SimCLR-100ep, SimCLR-300ep) per category. Results are **cached as JSON** — re-runs skip already-computed entries.

> ⚠️ **Critical:** PRO thresholds use **percentile-based spacing**, not linear. Raw PatchCore L2 distances are unnormalized and outlier values collapse linear-threshold PRO to near zero even when localization is visually correct.

### Phase 2 — Continual Learning

```
Notebook 4 (Naive baseline)  →  Notebook 5 (EWC | Replay | PackNet)
```

**Notebook 4 — Naive Baseline**
- Task stream: Bottle → Cable → Leather (300 epochs/task, SimCLR objective)
- Evaluates AUROC on **all seen categories** after each task
- Computes BWT, generates and saves forgetting curves
- State file enables Colab-disconnect-tolerant resumption

**Notebook 5 — CL Strategies**

Each strategy has its own independent state file:

```
results/phase2_ewc_state.json      # EWC: λ=100, Fisher from 20 batches/task
results/phase2_replay_state.json   # Coreset Replay: 100 imgs/cat, 32 replay/batch
results/phase2_packnet_state.json  # PackNet: 50% magnitude pruning per task
```

Strategies can be run independently in any order.

> **Why PackNet fails:** After 50% pruning per task × 3 tasks, only 12.5% of parameters remain free for Leather, confirmed by slow loss convergence (4.82→4.43 vs. 4.70→2.91 for Bottle without constraints). Magnitude-based pruning also misaligns with contrastive objectives — large-magnitude weights post-NT-Xent training do not correspond to importance for PatchCore's nearest-neighbor scoring.

## Methodology

### NT-Xent Contrastive Loss

$$\mathcal{L} = -\frac{1}{2N} \sum_{i=1}^{2N} \log \frac{\exp\!\bigl(\mathrm{sim}(z_i, z_j)/\tau\bigr)}{\displaystyle\sum_{k \neq i} \exp\!\bigl(\mathrm{sim}(z_i, z_k)/\tau\bigr)}$$

where $\mathrm{sim}(u,v) = u^\top v / (\|u\|\|v\|)$, $\tau = 0.5$, and the denominator sums over all $2N-1$ other samples in the batch.

### Backward Transfer (BWT)

$$\mathrm{BWT} = \frac{1}{T-1} \sum_{t=1}^{T-1} \bigl(R_{T,t} - R_{t,t}\bigr)$$

where $R_{T,t}$ is AUROC on task $t$ after training through all $T$ tasks, and $R_{t,t}$ is AUROC immediately after learning task $t$. Negative BWT = forgetting.

### EWC Regularized Loss

$$\mathcal{L}_\mathrm{total} = \mathcal{L}_\mathrm{NT\text{-}Xent} + \frac{\lambda}{2} \sum_i F_i \bigl(\theta_i - \theta_i^*\bigr)^2$$

Fisher diagonal $F_i$ approximated from NT-Xent gradients over 20 batches per completed task ($\lambda = 100$). Non-standard: $F_i$ quantifies each parameter's contribution to separating image pairs in projection space, not class log-likelihoods.

### Category-Aware Augmentation

```python
TEXTURE_CATEGORIES = {"leather", "carpet", "tile", "wood", "grid"}

def get_augmentation(category: str):
    is_texture = category.lower() in TEXTURE_CATEGORIES
    return SimCLRAugmentation(
        crop_scale_min        = 0.2 if is_texture else 0.5,
        # 0.5 → near-identical texture crops (weak negatives)
        # 0.2 → local grain structure (harder, more informative negatives)
        hue_jitter            = 0.0 if is_texture else 0.1,
        # Hue shift = defect signal in textures → disable entirely
        color_jitter_strength = 0.2,    # reduced from standard 0.8
        use_grayscale         = False,  # colour carries anomaly information
    )
```

### PatchCore Scoring

```
1. Extract features:  56×56×448  (layer1+layer2+layer3, bilinear upsample to 56×56)
2. Build coreset:     p_{k+1} = argmax_p  min_{m∈M} ||p − m||₂   → 10% of patches
3. Score test patch:  s(p) = min_{m∈M} ||p − m||₂
4. Image score:       S(x) = max_p s(p)
5. PRO thresholds:    percentile-spaced (not linear) to avoid outlier-scale collapse
```

## Known Bugs Fixed During Development

| Bug | Symptom | Fix Applied |
|-----|---------|-------------|
| `np.trapz` removed in NumPy 2.0 | `AttributeError` during PRO computation | `getattr(np, "trapezoid", None) or np.trapz` |
| Linear PRO thresholds | PRO ≈ 0 despite correct visual localization | Switched to percentile-based threshold spacing |
| Hue jitter on Leather | Color destruction → poor contrastive views | Set `hue=0` for all texture categories |
| Crop scale 0.5 on textures | Near-identical texture patches → contrastive collapse | Reduced to `crop_scale_min=0.2` for textures |
| Category argument mismatch in eval | Cable AUROC reported as 0.5063 (corrupted) | Fixed `score_test_set("bottle",...)` → `score_test_set("cable",...)` |
